## Project Overview

This is a **bird detection and classification system** that uses YOLOv8/v11 for object detection and HuggingFace models for species classification. The pipeline includes dataset preparation, model training, and inference on images/videos.

**Current trained models:**
- `model.pt` - PyTorch YOLOv8 model for bird detection (detects 7 species)
- `model.onnx` - ONNX-formatted version of the model for inference
- Config supports 400+ bird species (in `config.json`)

### Core Workflows

**1. Prepare Dataset (Auto-labeling)**


**3. Train Model**

**4. Run Detection on Images**

**5. Run Detection on Videos**

**6. Classify Bird Species (requires image with detected bird)**

## Architecture & Data Flow

### Main Components

1. **Data Preparation Pipeline**
   - `download_bird_images.py` - Fetches images from iNaturalist API (public, no key needed)
   - `auto_label.py` - Auto-generates bounding boxes using YOLOv8n detector (COCO bird class)
   - Detections filtered by confidence threshold (default 0.20)
   - Splits data: 90% train / 10% validation

2. **Training & Model Management**
   - `train_model.py` - Trains YOLOv11s on custom dataset
   - Uses YOLO format: `dataset/{images,labels}/{train,val}/`
   - Configuration: `data.yaml` (paths, class names, count)
   - Outputs best model to `runs/detect/train/weights/best.pt`

3. **Inference & Predictions**
   - `birds_detection.py` - Main detection script
     - `image_prediction()` - Detects birds in single image, annotates, saves
     - `video_prediction()` - Processes video frame-by-frame with ByteTrack tracking
     - Uses supervision library for visualization
     - Filters by confidence threshold

4. **Species Classification**
   - `classify_bird.py` - Classifies detected birds using HuggingFace EfficientNet-b2
   - Supports 400+ bird species from `config.json`
   - Accepts command-line args: `image`, `--top N`

5. **Quality Assurance**
   - `review_labels.py` - Visual QA tool for bounding boxes
   - Randomly samples labeled images
   - Draws boxes with colors per class, displays title

### Dataset Format

```
dataset/
├── <species_name>/          # Raw image folders
│   ├── bird1.jpg
│   ├── bird2.jpg
│   └── ...
├── images/
│   ├── train/              # 90% of images
│   └── val/                # 10% of images
├── labels/
│   ├── train/              # YOLO .txt format
│   └── val/
└── data.yaml               # Generated by auto_label.py
```

**YOLO Label Format** (`labels/train/image.txt`):
```
<class_id> <cx> <cy> <w> <h>
```
Where cx, cy, w, h are normalized (0-1) relative to image dimensions.

### Training Tips (from HOW_TO_TRAIN.md)

- **Adding new bird species**: Retraining from scratch (with yolo11s.pt) is recommended to avoid "catastrophic forgetting"
- **Fine-tuning existing model**: Only viable if class count stays exactly the same
- **Epochs**: Default 50 (adjust based on dataset size)
- **Image size**: Default 640 (must be multiple of 32)
- **Confidence threshold**: Inference uses 0.5 by default, can adjust in function calls

## Important Implementation Details

### Model Configuration
- `config.json` - EfficientNet-b2 config with 400+ bird species mapping (id2label)
- Currently only 7 species trained in `model.pt` (Crow, Kingfisher, Myna, Owl, Peacock, Pigeon, Sparrow)
- To expand: retrain with full config after collecting more data

### Detection Parameters
- `image_prediction()`:
  - Input: image path (local or URL)
  - `confidence` param: filters detections (default 0.5)
  - `model` param: path to .pt or .onnx file
  - Output: annotated image with boxes and confidence scores

- `video_prediction()`:
  - Requires `result_filename` to save output
  - Uses ByteTrack for object tracking across frames
  - Displays track IDs and class labels

### Auto-labeling (auto_label.py)
- Uses YOLOv8n detector to find birds (COCO class 14)
- Only keeps highest-confidence detection per image
- Skips images where no bird detected (logged to `unlabeled.txt`)
- Config: `CONF_THRESH=0.20`, `VAL_SPLIT=0.10`, `DETECTOR_MODEL="yolov8n.pt"`

## Common Development Tasks

**Experiment with different model sizes:**
- `yolo11n.pt` - nano (fastest)
- `yolo11s.pt` - small (default, balanced)
- `yolo11m.pt` - medium (slower, higher accuracy)
- Update `train_model.py` line 10

**Change auto-label confidence:**
- Edit `auto_label.py` line 22: `CONF_THRESH = 0.20`

**Adjust detection confidence at inference:**
- Edit `birds_detection.py` call: `image_prediction(..., confidence=0.5)`

**Switch between model formats:**
- `.pt` (PyTorch) - used for training
- `.onnx` (ONNX) - cross-platform inference (smaller, faster)

**Inspect model details:**
```bash
python inspect_model.py
```

## Troubleshooting

- **Out of GPU memory**: Reduce batch size or use smaller model (yolo11n vs yolo11m)
- **Auto-label skipping many images**: Lower `CONF_THRESH` in auto_label.py
- **Model not detecting birds**: Check confidence threshold is not too high; verify model file exists
- **Video processing slow**: Use smaller video resolution or reduce frame processing
